# 设计文档：GitHub README 展示

## 概述

本设计覆盖 FR-1 至 FR-5。交付物是中英文 README 和位于 `docs/images/` 的真实客户端截图，不修改运行时代码。

## 信息架构

```text
首屏定位与下载
  → 产品截图
  → 五分钟快速开始
  → ChatGPT / MCP 客户端接入
  → 核心能力与工具
  → 安全边界
  → 开发与贡献
```

## 技术方案

| 类别 | 选择 | 理由 | 关联需求 |
| --- | --- | --- | --- |
| 文档 | GitHub Flavored Markdown | GitHub 原生渲染，无额外站点依赖 | FR-1, FR-3, FR-5 |
| 图片 | PNG，相对路径 | 清晰、兼容 GitHub 和离线 clone | FR-2 |
| 截图来源 | 当前 Tauri 桌面客户端 | 避免设计稿或伪造状态与实际产品不一致 | FR-2, FR-4 |

## 文件结构

## 设计决策

### 决策 1：README 面向使用者优先（FR-1、FR-3）

仓库首页先回答“是什么、如何安装、如何连接”，内部架构和开发命令放在后半部分，避免用户先读实现细节。

### 决策 2：截图使用真实状态（FR-2）

使用真实桌面客户端窗口截图，优先选择不暴露敏感数据的总览、服务配置或连接指引区域；如界面包含敏感字段则通过调整页面状态避开，而不是制作虚假数据图。

### 决策 3：安全说明保持诚实（FR-4）

明确当前 Windows 子进程边界为 `policy_only`，不使用“完整沙箱”等超出实现的描述。

## 数据模型与 API

不涉及数据存储或 API 变更。

## 测试策略

- 检查中英文 README 的章节、链接和事实一致性。
- 检查所有相对图片路径存在且 Git 能跟踪。
- 人工查看 PNG，确认清晰度和敏感信息。
- 运行 Markdown 链接与 `git diff --check` 基础检查。

## 风险评估

| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| 截图泄露本机路径或密钥 | 高 | 选取安全页面并逐张人工审查 |
| README 与版本能力不一致 | 中 | 以 v0.1.6 实际 UI、Release 和测试结果为准 |
| 截图随 UI 更新过时 | 低 | 使用稳定总览页面并采用语义化文件名 |
