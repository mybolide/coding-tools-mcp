# 设计原则

## 1. Workspace First

Workspace 是产品的第一主对象，不是配置的附属项。用户打开应用首先看到项目卡片，而不是设置表单。

## 2. 渐进披露

默认只展示决策所需信息：名称、状态、endpoint。详细配置（OAuth 密钥、FRP 参数）在详情页按需展开，避免首屏表单堆叠。

## 3. 状态可感知

运行状态通过颜色、动画、文案三层传达：
- **颜色**：状态灯（绿/黄/灰/红）
- **动画**：运行中脉冲呼吸
- **文案**：「正在监听 127.0.0.1:28766」

## 4. 开发者友好

- Endpoint、路径、日志用等宽字体
- 一键复制是高频操作，按钮触手可及
- 错误信息包含可操作的修复建议

## 5. 克制的高级感

- 大量留白，信息密度低于传统后台
- 边框优于阴影（1px subtle border）
- 单一强调色，不滥用渐变
- 动效有目的（状态变化、页面过渡），不做装饰

## 6. 可访问性

- 深色/浅色双主题，跟随系统
- 正文对比度 ≥ 4.5:1
- 键盘可完整操作
- 支持 prefers-reduced-motion

## 应避免（反模式）

- 传统后台四宫格表单首页
- 嵌套卡片（card in card）
- 每段都有 EYEBROW 小标签
- emoji 作图标
- 无反馈的即时状态切换
